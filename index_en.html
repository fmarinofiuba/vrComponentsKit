VR Components Kit

This project proposes a set of high-level software components to facilitate the creation of immersive experiences. It focuses on solving common challenges faced by developers, such as XR controller management, dynamic user interface creation, scene navigation, selection and manipulation of virtual objects, and providing debugging tools.

The solution aims to abstractthe complexity of underlying APIs like WebXR and Three.js. Additionally, didactic examples are providedto functionally evaluate the modules, validate the components, and offer practical starting points for the development of web-based VR applications.

Examples

  1. Hello VR

    This is a basic example to initiate a VR scene.

  2. Navigation example

    This example evaluates the navigation module, specifically the "flight" mode and teleportation mechanism. It includes a scene with a colored ground plane and a 3D staircase model to test teleportation to different heights and verify incremental orientation changes without altering the user's position.

  3. Controllers example

    This example evaluates the correct functioning of the controller management module. It verifies the proper 3D representation of the controllers and the correct operation of event listening functions by printing events to the browser console.

  4. UILVRMenu example

    This example features a scene with two objects (Cube and TorusKnot) and a point light source. It defines a menu using the UILVrMenu class to control the rotation speeds of both objects, as well as the light's color and intensity. It also evaluates the menu's functionality in conjunction with the navigation module, ensuring that the controller's ray only activates UI actions when interacting with the menu, not teleportation.

  5. HTMLVRMenu example

    This example performs the same evaluations as the UILVRMenu example, but using an HTML-based menu.

  6. Debug Tools example

    In this example, a scene is created with a 3D teapot model that changes position following a sine wave. A VrVarsWatcher panel displays the teapot's position, rotation, and scale vectors in real-time. Additionally, a VrConsole panel is included to verify the correct display of simulated browser console messages in VR.

  7. Grabbable Objects example

    This example features a scene with both mobile and static objects. The first group of objects is configured for contact grabbing, while the second group uses "remoteGrabbing" mode. The example evaluates the correct functionality, especially when multiple objects are in the controller's ray, in which case only the closest object should be grabbed.

  8. Selectable Objects example

    In this example, a scene is created with five objects representing different materials and five objects representing various geometric shapes. A cylindrical pedestal in the center allows the user to select a material or shape, which then modifies the central object on the pedestal accordingly. The proposed objective was verified.
